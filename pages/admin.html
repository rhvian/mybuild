预警处置 · 建设中

v1.0 规划

该模块在 L2 阶段启用，用于把"质量 / 履约 / 安全 / 资金"四类风险点从数据层自动生成工单，并走派单→审批→反馈→关闭闭环。

  • 规则引擎：基于 quality_issue + 外部告警源生成 alerts
  • 工单表：派单 / 审批 / 备注 / 附件 / 关闭原因
  • 角色权限：审核员处置、管理员复核
  • 看板：按地区 / 类型 / 等级聚合

当前 v0.3 为数据采集演示版，业务闭环未开放。

联合惩戒管理 · 建设中

v1.0 规划

接入住建部 + 市场监管总局失信联合惩戒名录后上线，支持"限制招投标 / 重点监管 / 列入黑名单"的执行跟踪。

  • 数据源：信用中国 + 四库一平台诚信记录 + 省级黑名单
  • credit 实体类型：红黑名单时效 / 执行部门 / 解除条件
  • 跨部门协同：任务下发、回执、异议申诉

依赖数据主线 A4（黑名单接入）和编码主线 B5b（工单闭环）。

项目监管 · 建设中

v1.0 规划

采集到招投标完整数据（公告 / 中标 / 履约）后上线，针对重大工程做全周期留痕和监管分级。

  • 数据依赖：tender 数据深度从列表层升到招标+中标+履约
  • 重点项目标记：合同额 / 层级 / 风险等级
  • 监管日志：进度报送、质量抽检、争议处置

依赖数据主线 A5（招投标完整接入）。

用户与权限 · 建设中

v1.0 规划

真实用户体系在 L2 阶段随 FastAPI + JWT 后端同步上线，5 级角色：游客 / 招商 / 政务 / 审核员 / 管理员。

  • 表结构：users / roles / permissions / role_permissions
  • 认证：bcrypt + JWT（15min access + 7d refresh），登录失败限流
  • RBAC 中间件：require_role / require_permission
  • 审计：所有写操作落 audit_log 表

当前登录仅为前端口令校验，无后端 session。
